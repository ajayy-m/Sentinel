Sentinel Implementation Plan (Current State)
Date: 2026-05-04

============================================================
1) Objective
============================================================
Build a local-first, AI-assisted fleet monitor for small/medium lab fleets
(roughly 6-20 nodes), with human approval gates for all potentially unsafe
actions.

Current architecture is intentionally practical and already running:
- Agent -> Collector transport: ZeroMQ + msgpack
- Persistence: SQLite (hot operational store)
- UI: PyQt6 + pyqtgraph
- AI layers: anomaly scoring, root-cause hints, recommendations
- Discovery: dynamic LAN node/collector discovery over UDP broadcast

============================================================
2) Implemented Architecture (As-Built)
============================================================
Entry point
- sentinel.py
  - roles: agent, collector, report, ui, simulate

Agent
- sentinel/agent/agent.py
  - psutil-based collectors
  - stable node identity auto-derivation (hostname + digest)
  - explicit endpoint or auto-discovery fallback
  - periodic payload send to collector

Collector
- sentinel/core/collector.py
  - receives payloads on ZMQ PULL
  - runs health pipeline + anomaly scorer + root-cause + recommendations
  - stores artifacts in SQLite
  - tracks discovered nodes

Core
- sentinel/core/pipeline.py: threshold-based health scoring/alerts
- sentinel/core/anomaly.py: rolling z-score anomaly scoring
- sentinel/core/root_cause.py: heuristic correlator + optional LLM path
- sentinel/core/recommendations.py: recommendation generation (non-executing)
- sentinel/core/storage.py: append-focused persistence
- sentinel/core/schema.py: schema/protocol version validation
- sentinel/core/preflight.py: startup readiness checks (fail on critical)
- sentinel/core/identity.py: deterministic default node_id

Discovery
- sentinel/discovery.py
  - agent discovers collector
  - collector discovers agents

UI
- sentinel/ui/*
  - dashboard tabs for health, alerts, hints, recommendations, approvals,
    discovered nodes, and fleet trends
  - legacy Windows-classic visual style applied

Actions and Safety Gate
- sentinel/actions/gate.py
  - recommendation -> request workflow
  - explicit approve/reject records
  - non-executing safety design in current release

Docs delivered
- docs/SECURITY_BASELINE.md
- docs/ACTION_RUNBOOKS.md
- docs/AI_EVALUATION_KPI.md
- docs/DATA_LIFECYCLE_POLICY.md
- docs/PHASE0_HARDENING_SUMMARY.md
- docs/PACKAGING_DEPLOYMENT.md
